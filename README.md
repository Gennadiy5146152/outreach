# Outreach Desk

Локальный сервис для личного B2B-аутрича по ТЗ: Node.js, PostgreSQL, SMTP/IMAP, безопасная email-валидация, прогрев, open tracking, вложения и аналитика.

## Быстрый старт

1. Скопируйте настройки:

```bash
cp .env.example .env
```

2. Задайте логин и пароль для входа:

```dotenv
AUTH_USER=admin
AUTH_PASSWORD=сложный_пароль
AUTH_SESSION_SECRET=длинная_случайная_строка
```

Если сервис открыт через обычный `http://IP:3000`, оставьте `AUTH_COOKIE_SECURE=false`. Для HTTPS можно поставить `true`.

3. Запустите сервис:

```bash
docker compose up -d
```

4. Откройте:

```text
http://localhost:3000
```

По умолчанию включен безопасный dry-run: письма логируются и сохраняются в БД, но не уходят через SMTP. Для реальной отправки заполните SMTP/IMAP-настройки mailbox в UI и переключите режим в разделе `Настройки`.

## Что открывать первым

Откройте раздел `Старт`. Это главный экран для первичной настройки. Он показывает:

- что уже готово;
- что еще мешает запуску;
- куда нажать следующим шагом;
- включен ли безопасный dry-run режим.

Если непонятно, что делать: ничего не ищите по меню, просто откройте `Что делать` и нажимайте кнопку в блоке `Сейчас сделай это`.

На этом же экране есть блок `Что нужно заполнить в .env`: он показывает обязательные переменные, пароли mailbox и готовый шаблон строк для локального `.env`.

## Что значит каждый раздел

- `Старт` — пошаговый чеклист запуска.
- `Обзор` — главные цифры: лиды, отправки, открытия, ответы.
- `База` — добавление лидов, CSV-импорт, валидация email.
- `Почта` — подключение SMTP/IMAP, проверка доступов и SPF/DKIM/DMARC.
- `Прогрев` — прогревочные письма между вашими mailbox.
- `Кампании` — шаблоны, HTML-письма, вложения, preflight, тестовый запуск.
- `Очередь` — что будет отправлено, когда, прогресс, ETA и ошибки.
- `Входящие` — ответы из IMAP и ручная правка классификации.
- `Стоп-лист` — email и домены, которым нельзя писать.
- `События` — технический журнал действий.
- `Настройки` — текущий runtime: dry-run, tracking URL, лимиты вложений.

## Как начать прогрев и рассылки

### 1. Подключите mailbox

Откройте `Почта` и добавьте 2 ящика. Пароль почты вводится прямо в форме: сервис сам сохранит его в локальный `.env` в переменную вида `MAILBOX_NAME_DOMAIN_RU_PASSWORD`, а в базе оставит только имя этой переменной.

Для Яндекса обычно нужен пароль приложения, а не основной пароль аккаунта. Настройки:

```text
SMTP host: smtp.yandex.ru
SMTP port: 465
IMAP host: imap.yandex.ru
IMAP port: 993
Secure: true
```

После сохранения нажмите `Проверить`: сервис проверит SMTP, IMAP и MX/SPF/DKIM/DMARC домена.

### 2. Включите прогрев

В разделе `Прогрев` включите прогрев для обоих mailbox и нажмите `Отправить warmup сейчас`, либо дождитесь worker. Warmup отправляется только между вашими подключенными mailbox.

### 3. Подготовьте кампанию

1. Импортируйте или добавьте лидов.
2. Запустите валидацию.
3. Создайте кампанию.
4. Создайте шаги письма.
5. При необходимости загрузите вложения к шагу.
6. Добавьте valid/risky лидов в кампанию.
7. Запустите `Preflight`.
8. Сначала запустите `Тест`, затем `Старт manual` или `Старт auto`.

### 4. Включите реальную отправку

Когда тестовый сценарий прошел, откройте `Настройки` и переключите `Отправка` в режим `Реальная отправка`.

Runtime-настройки хранятся в PostgreSQL и применяются без пересоздания контейнеров.

### 5. Open tracking

Для реальных открытий нужен публичный URL, например через `ngrok` или `cloudflared`.

Укажите публичный адрес в `Настройки` → `Tracking URL`.

## Команды

```bash
npm run dev       # локальный backend без Docker
npm run worker    # локальный worker без Docker
npm run migrate   # применить миграции
npm run check     # проверка синтаксиса JS
npm run check:smoke # smoke-проверка запущенного сервиса
```

## Резервная копия PostgreSQL

```bash
docker compose exec postgres pg_dump -U outreach outreach > backup.sql
```

## Важное

- `.env` не коммитится.
- Вложения хранятся в `storage/attachments`.
- Open tracking для реальных получателей требует публичный Tracking URL, например через туннель.
- Стоп-лист доступен в UI: адреса и домены из него исключаются из коммуникации.
- Очередь показывает progress, ETA, countdown до следующего письма и ошибки отправки.
