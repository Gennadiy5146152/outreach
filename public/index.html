Что нужно заполнить в `.env`

Значения секретов здесь не показываются. Видно только, задана переменная или нет.

1

Подключить почту

Нужно добавить 2 своих ящика. Сервис проверит SMTP для отправки, IMAP для входящих и настройки домена.

Пароль вводится в форме Например: пароль приложения Яндекс Почты Сервис сам сохранит его в локальный .env
2

Включить прогрев

Прогрев отправляет письма только между твоими подключенными ящиками. Это проверяет отправку, входящие и ответы.

3

Добавить базу

Добавь лидов вручную или CSV, затем нажми “Запустить проверку email”. Невалидные адреса не попадут в отправку.

4

Сделать тестовую рассылку

Создай кампанию, добавь письмо, добавь лидов, нажми “Проверить перед запуском”, потом “Тест на мои почты”.

Очередь отправки

KPI

Новый лид

Минимум нужны компания и email. После добавления email автоматически попадет в очередь проверки.

Импорт и валидация

CSV должен содержать колонки `company` и `email`. После импорта нажми проверку email.

База

Клик по строке открывает карточку лида.

Подключить mailbox

SMTP нужен для отправки, IMAP — для входящих, прогрева и проверки ящика. Пароль можно ввести прямо здесь: сервис сохранит его в локальный `.env`, а в базе оставит только имя переменной.

Ящик
Отправка SMTP
Входящие IMAP

IMAP заполнять обязательно для проверки mailbox, входящих ответов и прогрева. Это не имя переменной `.env`: здесь указывается сервер и порт почты.

Доступы

После сохранения пароль будет записан в `.env` автоматически. В базе хранится только имя переменной, например `MAILBOX_NAME_DOMAIN_RU_PASSWORD`.
